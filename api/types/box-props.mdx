Props accepted by the `<Box>` component.

## Type Definition

```typescript
type BoxProps = Except<Styles, 'textWrap'> & {
  readonly 'aria-label'?: string;
  readonly 'aria-hidden'?: boolean;
  readonly 'aria-role'?:
    | 'button'
    | 'checkbox'
    | 'combobox'
    | 'list'
    | 'listbox'
    | 'listitem'
    | 'menu'
    | 'menuitem'
    | 'option'
    | 'progressbar'
    | 'radio'
    | 'radiogroup'
    | 'tab'
    | 'tablist'
    | 'table'
    | 'textbox'
    | 'timer'
    | 'toolbar';
  readonly 'aria-state'?: {
    readonly busy?: boolean;
    readonly checked?: boolean;
    readonly disabled?: boolean;
    readonly expanded?: boolean;
    readonly multiline?: boolean;
    readonly multiselectable?: boolean;
    readonly readonly?: boolean;
    readonly required?: boolean;
    readonly selected?: boolean;
  };
};
```

### Flexbox

<ResponseField name="flexGrow" type="number">
  This property defines the ability for a flex item to grow if necessary. See [flex-grow](https://css-tricks.com/almanac/properties/f/flex-grow/).
</ResponseField>

<ResponseField name="flexShrink" type="number">
  It specifies the "flex shrink factor", which determines how much the flex item will shrink relative to the rest of the flex items in the flex container when there isn't enough space on the row. See [flex-shrink](https://css-tricks.com/almanac/properties/f/flex-shrink/).
</ResponseField>

<ResponseField name="flexDirection" type="'row' | 'column' | 'row-reverse' | 'column-reverse'">
  It establishes the main-axis, thus defining the direction flex items are placed in the flex container. See [flex-direction](https://css-tricks.com/almanac/properties/f/flex-direction/).
</ResponseField>

<ResponseField name="flexBasis" type="number | string">
  It specifies the initial size of the flex item, before any available space is distributed according to the flex factors. See [flex-basis](https://css-tricks.com/almanac/properties/f/flex-basis/).
</ResponseField>

<ResponseField name="flexWrap" type="'nowrap' | 'wrap' | 'wrap-reverse'">
  It defines whether the flex items are forced in a single line or can be flowed into multiple lines. If set to multiple lines, it also defines the cross-axis which determines the direction new lines are stacked in. See [flex-wrap](https://css-tricks.com/almanac/properties/f/flex-wrap/).
</ResponseField>

<ResponseField name="alignItems" type="'flex-start' | 'center' | 'flex-end' | 'stretch' | 'baseline'">
  The align-items property defines the default behavior for how items are laid out along the cross axis (perpendicular to the main axis). See [align-items](https://css-tricks.com/almanac/properties/a/align-items/).
</ResponseField>

<ResponseField name="alignSelf" type="'flex-start' | 'center' | 'flex-end' | 'auto' | 'stretch' | 'baseline'">
  It makes possible to override the align-items value for specific flex items. See [align-self](https://css-tricks.com/almanac/properties/a/align-self/).
</ResponseField>

<ResponseField name="alignContent" type="'flex-start' | 'flex-end' | 'center' | 'stretch' | 'space-between' | 'space-around' | 'space-evenly'">
  It defines the alignment along the cross axis when there are multiple lines of flex items (when using flex-wrap). See [align-content](https://css-tricks.com/almanac/properties/a/align-content/).
</ResponseField>

<ResponseField name="justifyContent" type="'flex-start' | 'flex-end' | 'space-between' | 'space-around' | 'space-evenly' | 'center'">
  It defines the alignment along the main axis. See [justify-content](https://css-tricks.com/almanac/properties/j/justify-content/).
</ResponseField>

### Dimensions

<ResponseField name="width" type="number | string">
  Width of the element in spaces. You can also set it as a percentage, which will calculate the width based on the width of the parent element.
</ResponseField>

<ResponseField name="height" type="number | string">
  Height of the element in lines (rows). You can also set it as a percentage, which will calculate the height based on the height of the parent element.
</ResponseField>

<ResponseField name="minWidth" type="number | string">
  Sets a minimum width of the element. Percentages aren't supported yet; see https://github.com/facebook/yoga/issues/872.
</ResponseField>

<ResponseField name="minHeight" type="number | string">
  Sets a minimum height of the element in lines (rows). You can also set it as a percentage, which will calculate the minimum height based on the height of the parent element.
</ResponseField>

<ResponseField name="maxWidth" type="number | string">
  Sets a maximum width of the element. Percentages aren't supported yet; see https://github.com/facebook/yoga/issues/872.
</ResponseField>

<ResponseField name="maxHeight" type="number | string">
  Sets a maximum height of the element in lines (rows). You can also set it as a percentage, which will calculate the maximum height based on the height of the parent element.
</ResponseField>

<ResponseField name="aspectRatio" type="number">
  Defines the aspect ratio (width/height) for the element. Use it with at least one size constraint (`width`, `height`, `minHeight`, or `maxHeight`) so Ink can derive the missing dimension.
</ResponseField>

### Display

<ResponseField name="display" type="'flex' | 'none'">
