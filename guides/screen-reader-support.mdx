Ink has built-in support for screen readers, implementing a subset of the [ARIA specification](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA) to make your CLI applications accessible to users with visual impairments.

## Enabling Screen Reader Support

There are two ways to enable screen reader support in your Ink application:

### Option 1: Environment Variable

Set the `INK_SCREEN_READER` environment variable when running your app:

```bash
INK_SCREEN_READER=true my-cli
```

### Option 2: Render Option

Pass the `isScreenReaderEnabled` option to the `render()` function:

```jsx
import {render} from 'ink';

render(<MyApp />, {isScreenReaderEnabled: true});
```

## Detecting Screen Reader Mode

Use the `useIsScreenReaderEnabled` hook to detect if a screen reader is active:

```jsx
import {useIsScreenReaderEnabled, Text} from 'ink';

function MyComponent() {
  const isScreenReaderEnabled = useIsScreenReaderEnabled();
  
  return (
    <Text>
      {isScreenReaderEnabled
        ? 'Screen reader is enabled'
        : 'Screen reader is disabled'}
    </Text>
  );
}
```

This allows you to conditionally render different output for screen reader users.

## ARIA Attributes

Ink supports several ARIA attributes on `<Box>` and `<Text>` components:

### aria-label

Provide a custom label for screen readers:

```jsx
import {Box, Text} from 'ink';

function ProgressBar({progress}) {
  return (
    <Box>
      <Box width="50%" height={1} backgroundColor="green" />
      <Text aria-label={`Progress: ${progress}%`}>{progress}%</Text>
    </Box>
  );
}
```

Screen reader output: **"Progress: 50%"** instead of just "50%"

### aria-hidden

Hide decorative elements from screen readers:

```jsx
import {Box, Text} from 'ink';

function Message() {
  return (
    <Box>
      <Text aria-hidden>🎉 </Text>
      <Text>Congratulations!</Text>
    </Box>
  );
}
```

Screen readers will only read "Congratulations!" and skip the emoji.

### aria-role

Define the semantic role of an element. Supported roles:

- `button`
- `checkbox`
- `radio`
- `radiogroup`
- `list`
- `listitem`
- `menu`
- `menuitem`
- `progressbar`
- `tab`
- `tablist`
- `timer`
- `toolbar`
- `table`

```jsx
import {Box, Text} from 'ink';

function Button({label, onClick}) {
  return (
    <Box aria-role="button">
      <Text>{label}</Text>
    </Box>
  );
}
```

Screen reader output: **"button: {label}"**

### aria-state

Define the state of an element. Supported states:

- `checked` (boolean) - For checkboxes and radio buttons
- `disabled` (boolean) - For disabled elements
- `expanded` (boolean) - For expandable elements
- `selected` (boolean) - For selected elements

```jsx
import {Box, Text} from 'ink';
import {useState} from 'react';

function Checkbox({label}) {
  const [checked, setChecked] = useState(false);
  
  return (
    <Box aria-role="checkbox" aria-state={{checked}}>
      <Text>{checked ? '[x]' : '[ ]'} {label}</Text>
    </Box>
  );
}
```

Screen reader output: **"(checked) checkbox: {label}"** or **"(unchecked) checkbox: {label}"**

## Practical Examples

### Accessible Checkbox

Here's a complete example from Ink's source code:

```jsx
import React, {useState} from 'react';
import {render, Text, Box, useInput} from 'ink';

function AriaExample() {
  const [checked, setChecked] = useState(false);
  
  useInput(key => {
    if (key === ' ') {
      setChecked(!checked);
    }
  });
  
  return (
    <Box flexDirection="column">
      <Text>
        Press spacebar to toggle the checkbox. This example is best experienced
        with a screen reader.
      </Text>
      <Box marginTop={1}>
        <Box aria-role="checkbox" aria-state={{checked}}>
          <Text>{checked ? '[x]' : '[ ]'}</Text>
        </Box>
      </Box>
      <Box marginTop={1}>
        <Text aria-hidden>This text is hidden from screen readers.</Text>
      </Box>
    </Box>
  );
}

render(<AriaExample />);
```

### Accessible List

```jsx
import {Box, Text} from 'ink';

function TodoList({items}) {
  return (
    <Box flexDirection="column" aria-role="list">
      {items.map((item, index) => (
        <Box key={index} aria-role="listitem">
          <Text>{item.done ? '✓' : '○'}</Text>
          <Text> {item.text}</Text>
        </Box>
      ))}
    </Box>
  );
}
```

Screen reader output:
```
list:
  listitem: ✓ Buy groceries
  listitem: ○ Walk the dog
  listitem: ✓ Write documentation
```

### Accessible Menu

```jsx
import {Box, Text, useInput, useFocus} from 'ink';
import {useState} from 'react';

function MenuItem({label, isSelected}) {
  const {isFocused} = useFocus();
  
  return (
    <Box aria-role="menuitem" aria-state={{selected: isSelected}}>
      <Text color={isFocused ? 'cyan' : 'white'}>
        {isSelected ? '▶ ' : '  '}{label}
      </Text>
    </Box>
  );
}

function Menu({items}) {
  const [selected, setSelected] = useState(0);
  
  return (
    <Box flexDirection="column" aria-role="menu">
      {items.map((item, index) => (
        <MenuItem
          key={index}
          label={item}
          isSelected={index === selected}
        />
      ))}
    </Box>
  );
}
```

### Progress Indicator

```jsx
import {Box, Text} from 'ink';

function ProgressBar({value, max}) {
  const percentage = Math.round((value / max) * 100);
  const filled = Math.round((value / max) * 20);
  
  return (
    <Box aria-role="progressbar" aria-state={{value: percentage}}>
      <Box width={20} borderStyle="single">
        <Box width={filled} backgroundColor="green">
          <Text> </Text>
        </Box>
      </Box>
      <Text aria-label={`Progress: ${percentage}% complete`}>
        {' '}{percentage}%
      </Text>
    </Box>
  );
}
```

## Best Practices

<Tip>
  Always provide descriptive `aria-label` attributes for complex visual elements like progress bars, graphs, or spinners.
</Tip>

### Do's

- **Use semantic roles** for interactive elements (buttons, checkboxes, menus)
- **Provide descriptive labels** with `aria-label` for visual elements
- **Hide decorative content** with `aria-hidden` (icons, borders, ASCII art)
- **Test with screen readers** to ensure your output makes sense when read aloud
- **Provide text alternatives** for visual indicators

```jsx
// Good - descriptive for screen readers
<Box aria-role="button" aria-label="Save changes">
  <Text>💾 Save</Text>
</Box>

// Better - hide decorative emoji
<Box aria-role="button">
  <Text aria-hidden>💾 </Text>
  <Text>Save changes</Text>
</Box>
```

### Don'ts

- **Don't rely solely on visual cues** (colors, icons, formatting)
- **Don't hide important information** with `aria-hidden`
- **Don't use generic labels** like "Click here" or "Button"
- **Don't forget state changes** - update `aria-state` when UI state changes

```jsx
// Bad - no semantic meaning
<Box>
  <Text color="red">!</Text>
  <Text>Error</Text>
</Box>

// Good - provides role and context
<Box aria-role="alert">
  <Text aria-hidden color="red">! </Text>
  <Text>Error: File not found</Text>
</Box>
```

## Conditional Rendering for Screen Readers

Render different content based on screen reader mode:

```jsx
import {useIsScreenReaderEnabled, Box, Text} from 'ink';

function Spinner() {
  const isScreenReaderEnabled = useIsScreenReaderEnabled();
  
  if (isScreenReaderEnabled) {
    return <Text>Loading... Please wait.</Text>;
  }
  
  return (
    <Box>
      <Text>⠋ Loading...</Text>
    </Box>
  );
}
```

## Output Examples

When screen reader support is enabled, Ink transforms your UI into screen-reader-friendly text:

```jsx
<Box aria-role="checkbox" aria-state={{checked: true}}>
  <Text>Accept terms and conditions</Text>
</Box>
```

**Screen reader output:**
```
(checked) checkbox: Accept terms and conditions
```

```jsx
<Box aria-role="button" aria-state={{disabled: true}}>
  <Text>Submit</Text>
</Box>
```

**Screen reader output:**
```
(disabled) button: Submit
```

## Testing Accessibility

Test your app with screen reader support enabled:

```bash
INK_SCREEN_READER=true node my-cli.js
```

Or in your test suite:

```jsx
import {render} from 'ink';

const {lastFrame} = render(<MyComponent />, {
  isScreenReaderEnabled: true
});

// Verify screen reader output
expect(lastFrame()).toContain('checkbox:');
```

## Creating Accessible Components

<Steps>
  <Step title="Identify interactive elements">
    Determine which elements users interact with (buttons, inputs, checkboxes).
  </Step>
  
  <Step title="Add semantic roles">
    Use `aria-role` to define the element type:
    ```jsx
    <Box aria-role="button">Click me</Box>
    ```
  </Step>
  
  <Step title="Add states">
    Use `aria-state` for stateful elements:
    ```jsx
    <Box aria-role="checkbox" aria-state={{checked: true}} />
    ```
  </Step>
  
  <Step title="Provide labels">
    Use `aria-label` for descriptive context:
    ```jsx
    <Text aria-label="Loading: 45% complete">45%</Text>
    ```
  </Step>
  
  <Step title="Hide decorations">
    Use `aria-hidden` for visual-only elements:
    ```jsx
    <Text aria-hidden>═════════</Text>
    ```
  </Step>
</Steps>
